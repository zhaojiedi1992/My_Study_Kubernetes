生命周期回调
=============================



回调方式
-------------------

回调的生命周期可以在PostStart和PreStop进行配置的。

- PostStart: 在容器被创建之后立即被执行。 
- PreStop: 在容器因 API 请求或者管理事件（诸如存活态探针、启动探针失败、资源抢占、资源竞争等） 而被终止之前


具体处理程序实现
-------------------

- Exec: 执行一个脚本
- HTTP: 发送一个http请求


回调处理程序的日志不会在 Pod 事件中公开。 如果处理程序由于某种原因失败，它将播放一个事件。
